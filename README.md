Вариант №4

Основные свойства и достоинства:

Рекурсивный подход интуитивно понятен, хорошо подходит для целей обучения и иллюстрации принципов декомпозиции задач.​

Код на всех языках получается компактным, читаемым и легко объяснимым, особенно для задач, которые по своей природе делятся на повторяющиеся шаги.​

Так как реализация трёх языков схожа (различие в синтаксисе), данный алгоритм легко адаптировать и переносить между языками программирования.​

Практические недостатки:

Рекурсивный способ может приводить к переполнению стека при обработке длинных строк: глубина рекурсии линейна по количеству символов в строке.​

Производительность и эффективность уступают итеративным решениям, так как каждый вызов функции занимает место в стеке и увеличивает время выполнения.​

Для реальных задач с большими объёмами данных рекомендуется использовать итеративные методы (циклы или встроенные функции типа reverse в C++), чтобы избежать проблем с памятью и скоростью.​


Итог
Рекурсивные функции для переворота строки — отличный пример практической рекурсии, полезный для обучения и небольших задач. В производственном коде для работы с длинными или частыми строками стоит отдавать предпочтение более эффективным итеративным алгоритмам


Временная сложность рекурсивного алгоритма реверса строки составляет $$O(n^2)$$, где $$n$$ — длина строки.

### Объяснение сложности

- **Количество вызовов функции:** Каждый символ строки обрабатывается отдельным рекурсивным вызовом, то есть всего будет выполнено $$n$$ рекурсий (для строки из $$n$$ символов).
- **Склеивание строк:** На каждом уровне рекурсии происходит объединение одной буквы с результатом всех предыдущих вызовов. В большинстве языков (например, Python, Java, C++ с std::string) операция сложения/конкатенации строк занимает время, зависящее от их длины — также до $$O(n)$$ для каждой операции при стандартной реализации.
- **Итоговая оценка:** Из-за последовательного добавления символа к увеличивающейся строке на каждом уровне, итоговая сложность — квадратичная, т.е. $$O(n^2)$$.
